abstract: >-
  <span class="lang-en">Designing a lattice for a target vibration response is a backwards problem. Many shapes hit the same response, and checking any of them still means a finite-element solve. A usable design loop wants two networks, a quick forward one from shape to response and a generator that suggests shapes for a chosen response. Most published deep-learning work shrinks the response to a band-gap width and a center frequency. This poster trains the forward network on the whole band structure.<br><br>The model treats the unit cell as a graph. Junctions are the nodes, beams are the edges. Node and edge features go up to 768 dimensions and pass through six GINE message-passing layers, so a junction ends up seeing about six nodes out. Ten learned band queries then pool the nodes by attention, one per band, and hand back a vector for each band. A short residual stack of 1D convolutions along the wavevector axis mixes the bands and smooths the dispersion. Out come ten bands at 311 k-points along Y&Gamma;XM&Gamma;, each with a frequency and a three-way polarization split (one longitudinal, two transverse). A single pass does the job of the eigenvalue solve. The equations are:<br>$$h_i^{(l+1)} = \mathrm{ReLU}\!\big(\mathrm{BatchNorm}(\mathrm{GINEConv}(h^{(l)},e))\big) + h_i^{(l)}$$$$\mathrm{GINEConv}(h,e)_i = \mathrm{MLP}_\theta\!\Big((1{+}\epsilon)\,h_i + \!\!\sum_{j\in\mathcal{N}(i)}\!\! \mathrm{ReLU}(h_j + e_{j\to i})\Big)$$$$\alpha_{q,i} = \mathrm{softmax}_i\langle Q_q,\,h_i\rangle,\qquad z_q = \sum_i \alpha_{q,i}\,h_i$$$$\hat{y} = f_\theta(\mathcal{G})\in\mathbb{R}^{4\times 10\times 311}$$<br>Getting clean training data is the hard part: the simulator does not label the bands the same way from one structure to the next. To get around it, predicted and true bands are matched optimally first, with a fresh Hungarian assignment each step, and the error is measured only after that. The model is scored on the bands it gets, regardless of the order it lists them. A few extra terms on correlation, slope and amplitude along k stop the upper bands from going flat, and a second head reads polarization off the same match. The assignment and the loss come out as<br>$$\pi^\star = \arg\min_{\pi\in S_{10}} \sum_{j} \big\lVert \hat{y}_j - y_{\pi(j)}\big\rVert^2$$$$\mathcal{L} = \underbrace{\frac{1}{BK}\sum_{i,k} w_{\pi^\star(i)}\big(\hat{y}_{i,k} - y_{\pi^\star(i),k}\big)^2}_{\text{Magnitude}} + \lambda\,\underbrace{\big(\mathcal{L}_r + \mathcal{L}_{\partial_k} + \mathcal{L}_\sigma\big)}_{\text{Shape}}$$$$\mathcal{L}_r = 1 - \mathrm{corr}_k(\hat{\mathbf{y}},\mathbf{y}),\quad \mathcal{L}_{\partial_k} = \big\lVert \partial_k\hat{\mathbf{y}} - \partial_k\mathbf{y}\big\rVert^2,\quad \mathcal{L}_\sigma = \big(\sigma_k\hat{\mathbf{y}} - \sigma_k\mathbf{y}\big)^2$$<br>The next step is the other half. A graph VAE will run the map the other way, from a target band structure back to a geometry, and the same recipe should carry over to other physical properties.<br><br><strong>Glossary</strong><br>&bull; <strong>GNN / GINE</strong>&nbsp;: graph neural network; GINE is an edge-aware message-passing layer.<br>&bull; <strong>Message-passing</strong>&nbsp;: each node updates by gathering information from its neighbours.<br>&bull; <strong>Attention / band query</strong>&nbsp;: learned vectors that weight and pool the nodes into one embedding per band.<br>&bull; <strong>Residual stack</strong>&nbsp;: layers added back to their input, so they refine rather than replace it.<br>&bull; <strong>Hungarian matching</strong>&nbsp;: optimal one-to-one pairing of predicted and true bands, so the model is graded on the set of curves, not their order.<br>&bull; <strong>Forward / inverse model</strong>&nbsp;: predicts the response from a geometry / generates a geometry for a target response.<br>&bull; <strong>Graph VAE</strong>&nbsp;: generative model over graphs, here for inverse design.<br><br><strong>Notation</strong><br>&bull; \(\hat{y}\): Prediction<br>&bull; \(y\): Target<br>&bull; \(\pi^\star\): Optimal band permutation<br>&bull; \(B\): Batch size<br>&bull; \(K\): k-points per band<br>&bull; \(w_b\): Per-band weight<br>&bull; \(\lambda\): Weight of the shape terms<br>&bull; \(\partial_k\): Derivative along k<br>&bull; \(\sigma_k\): Standard deviation along k<br>&bull; \(\mathrm{corr}_k\): Correlation along k<br>&bull; \(Q_q\): Band query<br>&bull; \(\mathcal{N}(i)\): Neighbours of node \(i\)</span><span class="lang-fr">Concevoir un treillis pour une r&eacute;ponse vibratoire vis&eacute;e est un probl&egrave;me &agrave; l'envers.
