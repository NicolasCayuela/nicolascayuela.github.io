abstract: >-
  <span class="lang-en">Finding a lattice geometry that produces a wanted vibrational response is a one-to-many problem, and each candidate still has to go through a finite-element solve before you can check it. A workable design pipeline needs two networks. One maps a geometry to its response quickly; the other proposes geometries for a requested response. Much of the published deep-learning work keeps the problem small by reducing the response to a band-gap width and a center frequency. This poster covers the forward half, trained on the full band structure rather than those two numbers.<br><br>The model reads the unit cell as a graph: lattice junctions become nodes, beams become edges. Node and edge features are lifted to 768 dimensions and run through six GINE message-passing layers, which let each junction draw on information up to six nodes away. Ten learned band queries then pool the nodes by attention, one query per band, producing a vector per band. A short residual stack of 1D convolutions along the wavevector axis mixes the bands and smooths the dispersion. The result is ten bands at 311 k-points along Y&Gamma;XM&Gamma;, each carrying a frequency and a three-way polarization split (one longitudinal, two transverse). One pass replaces the eigenvalue solve. A GINE layer updates each node from its neighbours, ten band queries \(Q_q\) pool the nodes by attention, and the network outputs a \(4\times 10\times 311\) tensor (frequency + three polarization channels):<br>$$h_i^{(l+1)} = \mathrm{ReLU}\!\big(\mathrm{BatchNorm}(\mathrm{GINEConv}(h^{(l)},e))\big) + h_i^{(l)}$$$$\mathrm{GINEConv}(h,e)_i = \mathrm{MLP}_\theta\!\Big((1{+}\epsilon)\,h_i + \!\!\sum_{j\in\mathcal{N}(i)}\!\! \mathrm{ReLU}(h_j + e_{j\to i})\Big)$$$$\alpha_{q,i} = \mathrm{softmax}_i\langle Q_q,\,h_i\rangle,\qquad z_q = \sum_i \alpha_{q,i}\,h_i$$$$\hat{y} = f_\theta(\mathcal{G})\in\mathbb{R}^{4\times 10\times 311}$$<br>Preparing healthy data is difficult, label bands are not the same from one structure to the next. The fix is to match predicted and true bands optimally before measuring any error, with a Hungarian assignment recomputed at every step. The model is then graded on which curves it produced, not on the order it put them in. Extra terms on correlation, slope and amplitude along k keep the top bands from collapsing, and a second head reads polarization off the same match. The optimal band assignment \(\pi^\star\) and the matched loss are<br>$$\pi^\star = \arg\min_{\pi\in S_{10}} \sum_{j} \big\lVert \hat{y}_j - y_{\pi(j)}\big\rVert^2$$$$\mathcal{L} = \underbrace{\frac{1}{BK}\sum_{i,k} w_{\pi^\star(i)}\big(\hat{y}_{i,k} - y_{\pi^\star(i),k}\big)^2}_{\text{Magnitude}} + \lambda\,\underbrace{\big(\mathcal{L}_r + \mathcal{L}_{\partial_k} + \mathcal{L}_\sigma\big)}_{\text{Shape}}$$$$\mathcal{L}_r = 1 - \mathrm{corr}_k(\hat{\mathbf{y}},\mathbf{y}),\quad \mathcal{L}_{\partial_k} = \big\lVert \partial_k\hat{\mathbf{y}} - \partial_k\mathbf{y}\big\rVert^2,\quad \mathcal{L}_\sigma = \big(\sigma_k\hat{\mathbf{y}} - \sigma_k\mathbf{y}\big)^2$$<br>This forward model is the groundwork for the next piece: a graph VAE that runs the map backwards, from a target band structure to a geometry, with the same approach later extended to other physical properties.<br><br><strong>Glossary</strong><br>&bull; <strong>GNN / GINE</strong>&nbsp;: graph neural network; GINE is an edge-aware message-passing layer.<br>&bull; <strong>Message-passing</strong>&nbsp;: each node updates by gathering information from its neighbours.<br>&bull; <strong>Attention / band query</strong>&nbsp;: learned vectors that weight and pool the nodes into one embedding per band.<br>&bull; <strong>Residual stack</strong>&nbsp;: layers added back to their input, so they refine rather than replace it.<br>&bull; <strong>Hungarian matching</strong>&nbsp;: optimal one-to-one pairing of predicted and true bands, so the model is graded on the set of curves, not their order.<br>&bull; <strong>Forward / inverse model</strong>&nbsp;: predicts the response from a geometry / generates a geometry for a target response.<br>&bull; <strong>Graph VAE</strong>&nbsp;: generative model over graphs, here for inverse design.<br><br><strong>Notation</strong><br>&bull; \(\hat{y}\): Prediction<br>&bull; \(y\): Target<br>&bull; \(\pi^\star\): Optimal band permutation<br>&bull; \(B\): Batch size<br>&bull; \(K\): k-points per band<br>&bull; \(w_b\): Per-band weight<br>&bull; \(\lambda\): Weight of the shape terms<br>&bull; \(\partial_k\): Derivative along k<br>&bull; \(\sigma_k\): Standard deviation along k<br>&bull;
